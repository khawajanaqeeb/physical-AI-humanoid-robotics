# Curriculum Designer Skill

## Purpose
Design educational content structure, learning objectives, exercises, and assessment materials for the Physical AI & Humanoid Robotics textbook.

## Expertise Areas
- **Learning Progression**: Beginner to advanced topic sequencing
- **Pedagogical Design**: Effective teaching strategies for technical content
- **Learning Objectives**: Clear, measurable educational goals
- **Exercise Design**: Problems, labs, projects at appropriate difficulty levels
- **Assessment Creation**: Quizzes, review questions, practical assignments
- **Prerequisite Management**: Ensuring proper knowledge dependencies
- **Engagement Strategies**: Making technical content accessible and interesting

## When to Use This Skill
- Planning new textbook chapters or sections
- Creating learning objectives for topics
- Designing hands-on exercises and labs
- Developing assessment questions
- Structuring content for different skill levels
- Creating practice problems and solutions

## Content Design Process

### 1. Learning Objectives
Create SMART objectives (Specific, Measurable, Achievable, Relevant, Time-bound):
- "Students will be able to implement a PID controller for joint position control"
- "Students will understand the principles of bipedal locomotion and gait generation"
- "Students will analyze sensor fusion algorithms for robot state estimation"

### 2. Content Structure
- **Introduction**: Motivate the topic, real-world relevance
- **Fundamentals**: Core concepts and theory
- **Deep Dive**: Advanced topics and mathematical details
- **Examples**: Worked examples and case studies
- **Hands-On**: Practical exercises and simulations
- **Summary**: Key takeaways and review
- **Further Exploration**: Advanced topics and research directions

### 3. Exercise Types
- **Conceptual Questions**: Test understanding of theory
- **Mathematical Problems**: Apply equations and derivations
- **Coding Exercises**: Implement algorithms and systems
- **Simulation Labs**: Hands-on experimentation
- **Design Challenges**: Open-ended engineering problems
- **Research Assignments**: Literature review and analysis

## Output Format

### Chapter Structure
```
Chapter X: [Topic Name]

Learning Objectives:
1. [Objective 1]
2. [Objective 2]
3. [Objective 3]

Prerequisites:
- [Required prior knowledge]

Sections:
1. Introduction & Motivation
2. Fundamental Concepts
3. Technical Details
4. Implementation & Examples
5. Advanced Topics
6. Summary & Review

Exercises:
- [5-10 exercises of varying difficulty]

Assessment:
- Review questions
- Practical assignments
```

## Difficulty Levels
- **Beginner**: Conceptual understanding, basic calculations
- **Intermediate**: Implementation, algorithm application
- **Advanced**: Novel design, research-level challenges
- **Expert**: Cutting-edge topics, open problems

## Assessment Guidelines
- Align questions with learning objectives
- Include both recall and application questions
- Provide clear rubrics for open-ended problems
- Offer hints and solutions for self-study
- Include real-world scenarios and case studies
