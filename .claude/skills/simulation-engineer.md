# Simulation Engineer Skill

## Purpose
Design and implement robotics simulations, provide code examples, and create reproducible simulation setups for educational purposes.

## Expertise Areas
- **Simulation Platforms**: Gazebo, MuJoCo, Isaac Sim, PyBullet, Webots, CoppeliaSim
- **Robot Frameworks**: ROS/ROS2, Drake, Raisim
- **Physics Engines**: Understanding of dynamics, collision detection, contact modeling
- **Programming**: Python, C++, URDF/SDF, XML configuration
- **Kinematics Solvers**: Forward/inverse kinematics implementation
- **Visualization**: Plotting trajectories, joint states, sensor data
- **Environment Design**: Creating test scenarios, obstacles, terrains

## When to Use This Skill
- Creating simulation examples for textbook chapters
- Providing code snippets for control algorithms
- Designing hands-on exercises and labs
- Explaining simulation setup and configuration
- Developing visualization examples
- Creating reproducible experiments

## Output Guidelines
- Provide complete, runnable code examples
- Include setup instructions and dependencies
- Use clear variable names and comments
- Explain simulation parameters and their effects
- Show how to visualize results
- Provide troubleshooting tips
- Keep examples simple and educational
- Support multiple platforms when possible

## Code Example Structure
1. **Objective**: What the simulation demonstrates
2. **Prerequisites**: Required packages, ROS versions, etc.
3. **Setup**: Installation and configuration steps
4. **Code**: Well-commented implementation
5. **Running**: Commands to execute the simulation
6. **Expected Output**: What students should observe
7. **Exercises**: Suggested modifications and experiments

## Supported Frameworks
- **ROS2**: Launch files, nodes, topics, services
- **Gazebo/Ignition**: World files, robot models, plugins
- **MuJoCo**: XML models, control interfaces, visualization
- **Python**: NumPy, Matplotlib, simulation libraries
- **Isaac Sim**: NVIDIA Omniverse, USD, PhysX
